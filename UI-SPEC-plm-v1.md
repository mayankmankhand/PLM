## Accessibility Requirements
- Text on background: minimum 4.5:1 contrast ratio (WCAG AA)
- Interactive elements: visible focus indicators (outline or ring)
- Touch targets: minimum 44x44px
- Images: descriptive alt text
- Icon-only buttons: aria-label attribute

## UX Rules Checklist

### CRITICAL
- [ ] `color-contrast` (CRITICAL) - Maintain minimum 4.5:1 contrast ratio between text and background
- [ ] `focus-states` (CRITICAL) - All interactive elements must have a visible focus indicator for keyboard navigation
- [ ] `touch-targets` (CRITICAL) - Interactive elements must be at least 44x44px on touch devices
- [ ] `aria-labels` (CRITICAL) - Icon-only buttons and links must have accessible labels

### HIGH
- [ ] `font-size-minimum` (HIGH) - Body text must be at least 16px on mobile devices
- [ ] `line-height` (HIGH) - Body text line height should be 1.5 to 1.75 for readability
- [ ] `line-length` (HIGH) - Limit line length to 65-75 characters for comfortable reading
- [ ] `no-horizontal-scroll` (HIGH) - Pages must not scroll horizontally on any supported viewport
- [ ] `loading-states` (HIGH) - Show loading feedback for any operation taking more than 300ms
- [ ] `error-placement` (HIGH) - Display error messages near the element that caused the problem
- [ ] `color-not-only-indicator` (HIGH) - Never use color as the sole means of conveying information

### MEDIUM
- [ ] `cursor-pointer` (MEDIUM) - All clickable elements should show a pointer cursor on hover
- [ ] `consistent-spacing` (MEDIUM) - Use a consistent spacing scale throughout the interface
- [ ] `responsive-breakpoints` (MEDIUM) - Design for standard breakpoints and test at each one
- [ ] `animation-duration` (MEDIUM) - Keep UI animations between 150ms and 300ms
- [ ] `disable-during-async` (MEDIUM) - Disable buttons during asynchronous operations to prevent duplicate submissions
- [ ] `no-emoji-icons` (MEDIUM) - Use SVG icons instead of emoji for UI elements
- [ ] `consistent-icon-sizing` (MEDIUM) - All icons within the same context should use the same dimensions

## Technical Setup

### Tailwind Config
```js
// Add to tailwind.config.js > theme > extend
{
  colors: {
    primary: '#2563EB',
    secondary: '#475569',
    accent: '#7C3AED',
    background: '#FFFFFF',
    surface: '#F8FAFC',
    text: '#0F172A',
    'text-muted': '#64748B',
    border: '#E2E8F0',
    success: '#16A34A',
    warning: '#D97706',
    danger: '#DC2626',
  },
  fontFamily: {
    heading: ['Inter', '-apple-system', 'BlinkMacSystemFont', 'sans-serif'],
    body: ['Inter', '-apple-system', 'BlinkMacSystemFont', 'sans-serif'],
  }
}
```

### CSS Variables
```css
:root {
  /* Colors */
  --color-primary: #2563EB;
  --color-secondary: #475569;
  --color-accent: #7C3AED;
  --color-background: #FFFFFF;
  --color-surface: #F8FAFC;
  --color-text: #0F172A;
  --color-text-muted: #64748B;
  --color-border: #E2E8F0;
  --color-success: #16A34A;
  --color-warning: #D97706;
  --color-danger: #DC2626;

  /* Typography */
  --font-heading: 'Inter', -apple-system, BlinkMacSystemFont, sans-serif;
  --font-body: 'Inter', -apple-system, BlinkMacSystemFont, sans-serif;
}
```

**Note:** Use Tailwind classes with these semantic color names instead of hardcoding hex values. This keeps your design consistent and makes future changes easy - update one token and it changes everywhere.
