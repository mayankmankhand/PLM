## 1. Design Tokens - Color Palette

Warm off-white palette replacing the original cold SaaS palette.

| Role | Hex | Usage |
|------|-----|-------|
| Background | `#FAF9F7` | Page background, main canvas |
| Surface | `#F5F4F0` | Cards, panels, grouped sections |
| Surface Elevated | `#FFFFFF` | Message containers, input fields, elevated elements |
| Primary | `#2563EB` | Actions, links, active states, focus rings |
| Primary Soft | `#EFF6FF` | Light primary backgrounds, avatar containers |
| Secondary | `#57534E` | Supporting elements, secondary buttons |
| Accent | `#7C3AED` | Highlights, special callouts |
| Text | `#1C1917` | Body text, headings |
| Text Muted | `#78716C` | Captions, labels, placeholders |
| Border | `#E7E5E4` | Input borders, card borders, dividers |
| Border Subtle | `#F5F5F4` | Very faint section dividers |
| Success | `#16A34A` | Confirmations, positive states |
| Warning | `#D97706` | Caution messages, pending states |
| Danger | `#DC2626` | Errors, destructive actions |

### WCAG AA Contrast Verification

All text/background combinations meet WCAG AA (minimum 4.5:1 for normal text).

| Combination | Ratio | Verdict |
|-------------|-------|---------|
| Text `#1C1917` on Background `#FAF9F7` | ~15.5:1 | PASS |
| Text `#1C1917` on Surface Elevated `#FFFFFF` | ~16.7:1 | PASS |
| Text `#1C1917` on Surface `#F5F4F0` | ~14.8:1 | PASS |
| Text Muted `#78716C` on Background `#FAF9F7` | ~4.9:1 | PASS |
| Text Muted `#78716C` on Surface `#F5F4F0` | ~4.5:1 | PASS (threshold) |
| Text Muted `#78716C` on Surface Elevated `#FFFFFF` | ~5.2:1 | PASS |

> Update this table if any token changes. Muted text on Surface is at the 4.5:1 threshold - do not darken Surface further.

## 2. Typography

Font: **Inter** (loaded via `next/font/google`, applied as CSS variable).

| Role | Weight | Size | Line Height | Usage |
|------|--------|------|-------------|-------|
| Heading | 600 (semibold) | 18-24px | 1.4 | Section titles, panel headers |
| Body | 400 (regular) | 15px | 1.625 (26px) | Chat messages, descriptions |
| Small/Caption | 500 (medium) | 12px | 1.5 | Labels, timestamps, field names |
| Code | mono stack | 14px | 1.5 | Code blocks, identifiers |

Monospace stack: `ui-monospace, SFMono-Regular, 'SF Mono', Menlo, Consolas, monospace`

Notes:
- `inter.variable` goes on `<html>` tag for Tailwind v4 @theme to resolve `var(--font-inter)`
- Body at 15px (not 16px) matches the chat-optimized density we want
- Mobile minimum: 15px body, 12px captions

## 3. Status Badge System

Six lifecycle states rendered as pill badges. Color is always paired with a text label - never the only indicator.

| Status | Background | Text Color | Border | Style |
|--------|-----------|------------|--------|-------|
| DRAFT | `#FEF3C7` (amber-100) | `#92400E` (amber-800) | none | filled |
| PUBLISHED | `#DCFCE7` (green-100) | `#166534` (green-800) | none | filled |
| OBSOLETE | `#F5F5F4` (stone-100) | `#78716C` (stone-500) | none | filled, dimmed |
| PENDING | `transparent` | `#78716C` (stone-500) | `1px solid #D6D3D1` (stone-300) | outlined/hollow |
| PASSED | `#DCFCE7` (green-100) | `#166534` (green-800) | none | filled |
| FAILED | `#FEE2E2` (red-100) | `#991B1B` (red-800) | none | filled |

### Badge Specs

- Font size: 11px
- Font weight: 500 (medium)
- Text transform: uppercase
- Letter spacing: tracking-wide (0.05em)
- Padding: 2px 8px
- Border radius: 9999px (full pill)
- No icons inside badges - text only

## 5. Component Styling

**User messages:**
- Background: Surface Elevated `#FFFFFF`
- Border: 1px solid Border `#E7E5E4`
- Border radius: rounded-2xl
- Padding: 16px

**Assistant messages:**
- Background: transparent (no container)
- Text color: Text `#1C1917`
- Just text content, no wrapper box

**Avatar:**
- Size: 28px
- Border radius: rounded-lg
- Background: Primary Soft `#EFF6FF`
- Icon color: Primary `#2563EB`

**Sender label:**
- Font size: 12px
- Color: Text Muted `#78716C`
- Font weight: 500 (medium)

### Tool Indicators

Full-width inline blocks (not tiny badges).

| State | Background | Text | Icon | Behavior |
|-------|-----------|------|------|----------|
| Active | Primary Soft `#EFF6FF` | Primary `#2563EB` | Animated pulse dot (left) | Expanded, shows tool name |
| Completed | transparent | Success `#16A34A` | Static checkmark | Collapses to single line |
| Error | `#FEF2F2` (red-50) | Danger `#DC2626` | Alert triangle | Expanded, shows error summary |

- Font size: 13px
- Font weight: 500 (medium)

### Chat Input (Composer)

- Background: Surface Elevated `#FFFFFF`
- Border: 1px solid Border `#E7E5E4`
- Shadow: `0 -1px 3px rgba(0,0,0,0.04)` (subtle upward shadow)
- Border radius: rounded-xl
- Placeholder color: Text Muted `#78716C`

### Context Panel

- Background: Surface `#F5F4F0`
- Header background: Surface Elevated `#FFFFFF` with bottom border
- Width: 480px (desktop, side-by-side)
- Mobile (below 768px): full-width slide-over drawer with backdrop

### Detail View

- Grouped sections separated by Border Subtle `#F5F5F4` dividers
- Status fields render as badges (see Section 3)
- Field labels: 12px, Text Muted `#78716C`, uppercase, tracking-wide (0.05em)
- Field values: 14px, Text `#1C1917`, normal weight (400)

### Table View

- Header row: Surface `#F5F4F0` background, 12px uppercase Text Muted
- Data rows: Surface Elevated `#FFFFFF` background
- Row hover: slightly darker (`#FAFAF9`)
- Status cells: render as badges (see Section 3)
- Empty state: oversized lucide icon at 25-30% opacity, warm heading text below

### Empty/Welcome State

- Large lucide icon (48-64px) at 25-30% opacity
- Heading: Text `#1C1917`, semibold
- Subtext: Text Muted `#78716C`
- Suggestion chips: Surface Elevated `#FFFFFF` background, Border `#E7E5E4` border, hover adds subtle shadow lift

## 6. Motion and Animation

| Animation | Duration | Easing | Details |
|-----------|----------|--------|---------|
| Panel slide | 200ms | ease-out | Slide in from right |
| Hover transitions | 150ms | ease | Background color, shadow changes |
| Shimmer (streaming) | 1.5s | linear, infinite | Subtle gradient sweep left-to-right |
| Streaming cursor | pulse | - | 1.5px wide, Primary `#2563EB` color |
| Streaming indicator | - | - | min-height 24px to prevent CLS |

### Reduced Motion

All animations respect `prefers-reduced-motion: reduce`:
- Durations collapse to 0ms
- Shimmer and pulse animations stop
- Panel appears instantly (no slide)
- Streaming indicator still reserves space (min-height stays)

## 7. Accessibility Requirements

### Critical
- Color contrast: WCAG AA minimum 4.5:1 (verified in Section 1)
- Focus indicators: 2px solid Primary `#2563EB` with 2px offset on all interactive elements
- Touch targets: minimum 44x44px
- Icon-only buttons: `aria-label` required
- Color is never the sole indicator (badges always have text labels)

### High
- Body text minimum 15px (mobile and desktop)
- Line height 1.5-1.75 for body text
- Line length capped at 65-75 characters (`max-w-4xl`)
- No horizontal scroll on any viewport
- Loading feedback for operations over 300ms
- Error messages placed near the triggering element

### Medium
- Pointer cursor on all clickable elements
- Consistent spacing scale (Section 4)
- Responsive breakpoints: 768px (tablet), 1024px (desktop)
- Animations 150-300ms (Section 6)
- Buttons disabled during async operations
- SVG icons only (no emoji as UI elements)
- Consistent icon sizing within context
- Reduced-motion support for all animations (Section 6)

## 8. Technical Setup

### Tailwind CSS v4 @theme Block

Add to `globals.css` inside the `@theme` directive:

```css
@theme {
  --color-primary: #2563EB;
  --color-primary-soft: #EFF6FF;
  --color-secondary: #57534E;
  --color-accent: #7C3AED;
  --color-background: #FAF9F7;
  --color-surface: #F5F4F0;
  --color-surface-elevated: #FFFFFF;
  --color-text: #1C1917;
  --color-text-muted: #78716C;
  --color-border: #E7E5E4;
  --color-border-subtle: #F5F5F4;
  --color-success: #16A34A;
  --color-warning: #D97706;
  --color-danger: #DC2626;
}
```

This makes tokens available as Tailwind utilities: `bg-background`, `text-text-muted`, `border-border`, etc.

### Usage Rules
- Always use semantic token names in Tailwind classes (e.g., `bg-surface`, `text-text-muted`)
- Never hardcode hex values in component files
- Update the WCAG table in Section 1 if any token value changes
