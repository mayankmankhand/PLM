## 1. Design Tokens - Color Palette

Warm earthy beige palette. Background is visibly warm (not subtle off-white). Primary shifted from cold blue to rich amber. Status badges use warm-tinted semantic colors.

| Role | Hex | Usage |
|------|-----|-------|
| Background | `#F8F0E3` | Page background, main canvas - visible warm beige |
| Surface | `#F0E9DC` | Cards, panels, grouped sections - deeper warm beige |
| Surface Elevated | `#FFFFFF` | Message containers, input fields, elevated elements |
| Primary | `#B45309` (amber-700) | Buttons, CTAs, active states, primary actions |
| Primary Soft | `#FEF3C7` (amber-100) | Light primary backgrounds, avatar containers |
| Secondary | `#57534E` (stone-600) | Supporting elements, secondary buttons |
| Accent | `#0369A1` (sky-700) | Links, focus rings, info callouts, diagram highlights |
| Text | `#292524` (stone-800) | Body text, headings |
| Text Muted | `#6D6560` | Captions, labels, placeholders |
| Border | `#D6D3D1` (stone-300) | Input borders, card borders, dividers |
| Border Subtle | `#E7E5E4` (stone-200) | Faint section dividers |
| Success | `#16A34A` (green-600) | Confirmations, positive states |
| Warning | `#D97706` (amber-600) | Caution messages, pending states |
| Danger | `#DC2626` (red-600) | Errors, destructive actions |

### What Changed from v1 Palette

| Token | Old | New | Why |
|-------|-----|-----|-----|
| Background | `#FAF9F7` | `#F8F0E3` | Old was barely distinguishable from white. New is visibly warm beige. |
| Surface | `#F5F4F0` | `#F0E9DC` | Deeper beige to maintain contrast with background |
| Primary | `#2563EB` (blue) | `#B45309` (amber) | Cold blue clashed with warm beige. Amber is the natural fit. |
| Primary Soft | `#EFF6FF` (blue tint) | `#FEF3C7` (amber tint) | Matches new primary |
| Accent | `#7C3AED` (violet) | `#0369A1` (sky) | Cool blue accent gives contrast against warm base |
| Text | `#1C1917` | `#292524` | Slightly warmer dark from stone scale (blog palette) |
| Text Muted | `#78716C` | `#6D6560` | Darkened slightly to maintain WCAG AA on warmer background |
| Border | `#E7E5E4` | `#D6D3D1` | Stronger border for visibility on warm background |
| Border Subtle | `#F5F5F4` | `#E7E5E4` | Previous was invisible. Now faintly visible. |

### WCAG AA Contrast Verification

All text/background combinations meet WCAG AA (minimum 4.5:1 for normal text).

| Combination | Ratio | Verdict |
|-------------|-------|---------|
| Text `#292524` on Background `#F8F0E3` | ~12.5:1 | PASS |
| Text `#292524` on Surface `#F0E9DC` | ~11.2:1 | PASS |
| Text `#292524` on Surface Elevated `#FFFFFF` | ~15.4:1 | PASS |
| Text Muted `#6D6560` on Background `#F8F0E3` | ~5.0:1 | PASS |
| Text Muted `#6D6560` on Surface `#F0E9DC` | ~4.7:1 | PASS |
| Text Muted `#6D6560` on Surface Elevated `#FFFFFF` | ~5.7:1 | PASS |
| Primary `#B45309` on Surface Elevated `#FFFFFF` | ~5.9:1 | PASS |
| Accent `#0369A1` on Surface Elevated `#FFFFFF` | ~5.7:1 | PASS |
| Accent `#0369A1` on Background `#F8F0E3` | ~4.8:1 | PASS |
| Accent `#0369A1` on Surface `#F0E9DC` | ~4.5:1 | PASS (threshold) |

> Update this table if any token changes. Muted text on Surface is near the 4.5:1 threshold - do not darken Surface further without re-checking.

## 3. Status Badge System

Six lifecycle states rendered as pill badges. Color is always paired with a text label - never the only indicator.

**Strategy:** Badges use warm-tinted versions of semantic colors so they feel part of the earthy palette instead of clashing with raw Tailwind greens and reds.

| Status | Background | Text Color | Border | Style |
|--------|-----------|------------|--------|-------|
| DRAFT | `#FEF3C7` (amber-100) | `#92400E` (amber-800) | none | filled - warm amber |
| APPROVED | `#E6F2E0` | `#3B6B35` | none | filled - warm sage green |
| CANCELED | `#F0E9DC` (surface) | `#6D6560` (text-muted) | none | filled, dimmed - uses palette tokens |
| PENDING | `transparent` | `#6D6560` (text-muted) | `1px solid #D6D3D1` (border) | outlined - uses palette tokens |
| PASSED | `#E6F2E0` | `#3B6B35` | none | filled - same as APPROVED |
| FAILED | `#F5E0D5` | `#9B3030` | none | filled - warm terra cotta red |

### What Changed from v1 Badges

| Status | What changed | Why |
|--------|-------------|-----|
| APPROVED/PASSED | `#DCFCE7`/`#166534` to `#E6F2E0`/`#3B6B35` | Old was cold mint green. New is warm sage that sits naturally on beige. |
| CANCELED | Raw `stone-100` to `Surface` token | Now uses the palette directly instead of a separate color. |
| PENDING | Raw `stone-500`/`stone-300` to palette tokens | Uses `Text Muted` and `Border` tokens. |
| FAILED | `#FEE2E2`/`#991B1B` to `#F5E0D5`/`#9B3030` | Old was cold pink-red. New has warm peach undertone. |
| DRAFT | No change | Already amber-based, matches the warm palette perfectly. |

### Badge Specs

- Font size: 11px
- Font weight: 500 (medium)
- Text transform: uppercase
- Letter spacing: tracking-wide (0.05em)
- Padding: 2px 8px
- Border radius: 9999px (full pill)
- No icons inside badges - text only

### Token Architecture

**Badge colors are component-local, not global tokens.** The status badge hex values (e.g., `#E6F2E0`, `#3B6B35`) live in the StatusBadge component, not in `@theme`. Rationale: these colors are only used in one place. If a color appears in 2+ unrelated components, promote it to a global token.

**PASSED and APPROVED share the same green intentionally.** Both represent a positive outcome in different domains (test execution vs requirement lifecycle). Same visual treatment is correct.

## 4. Color Usage Rules

### Primary vs Accent

| Role | Token | Use for |
|------|-------|---------|
| Primary (`#B45309`) | `bg-primary`, `text-primary` | Buttons, CTAs, submit actions, active tab indicators |
| Accent (`#0369A1`) | `text-accent`, `ring-accent` | Text links, focus rings, info callouts, diagram highlights |

**Why two colors?** Primary (amber) drives action. Accent (blue) signals "clickable" and "informational" - users expect blue for links and focus indicators. Using amber for focus rings would feel unfamiliar.

### Interaction States

Lightweight derivation rules instead of a full token matrix:

| State | Rule | Example |
|-------|------|---------|
| Hover | Darken 5-8% | Primary button bg darkens slightly |
| Active/Pressed | Darken 10-12% | Momentary press feedback |
| Disabled | Use `text-text-muted` + `bg-surface` + `opacity-60` | Clearly inactive, not just faded |
| Selected | Use `bg-primary-soft` + `text-primary` | Active tab, selected row |
| Focus | 2px solid Accent `#0369A1` with 2px offset | Keyboard navigation ring |

### Elevated Surface Rule

All `Surface Elevated` (`#FFFFFF`) elements must have `border-border` when placed on Background or Surface. Pure white without a border looks like it's floating on warm beige.

## 6. Component Styling

### Chat Messages (flat, left-aligned)

**User messages:**
- Background: Surface `#F0E9DC` (warm tint, not white card)
- Border: none (no border - blends with warm palette)
- Border radius: rounded-2xl
- Padding: 16px

**Assistant messages:**
- Background: transparent (no container)
- Text color: Text `#292524`

- Size: 28px
- Border radius: rounded-lg
- Background: Primary Soft `#FEF3C7`
- Icon color: Primary `#B45309`

**Sender label:**
- Font size: 12px
- Color: Text Muted `#6D6560`
- Font weight: 500 (medium)

### Tool Indicators

Full-width inline blocks (not tiny badges).

| State | Background | Text | Icon | Behavior |
|-------|-----------|------|------|----------|
| Active | Primary Soft `#FEF3C7` | Primary `#B45309` | Animated pulse dot (left) | Expanded, shows tool name |
| Completed | transparent | Success `#16A34A` | Static checkmark | Collapses to single line |
| Error | `#F5E0D5` (warm red tint) | Danger `#DC2626` | Alert triangle | Expanded, shows error summary |

- Font size: 13px
- Font weight: 500 (medium)

### Chat Input (Composer)

- Background: Surface Elevated `#FFFFFF`
- Border: 1px solid Border `#D6D3D1`
- Shadow: `0 -1px 3px rgba(0,0,0,0.04)` (subtle upward shadow)
- Border radius: rounded-xl
- Placeholder color: Text Muted `#6D6560`

### Context Panel

- Background: Surface `#F0E9DC`
- Header background: Surface Elevated `#FFFFFF` with bottom border
- Width: 480px (desktop, side-by-side)
- Mobile (below 768px): full-width slide-over drawer with backdrop

### Detail View

- Grouped sections separated by Border Subtle `#E7E5E4` dividers
- Status fields render as badges (see Section 3)
- Field labels: 12px, Text Muted `#6D6560`, uppercase, tracking-wide (0.05em)
- Field values: 14px, Text `#292524`, normal weight (400)

### Table View

- Header row: Surface `#F0E9DC` background, 12px uppercase Text Muted
- Data rows: Surface Elevated `#FFFFFF` background
- Row hover: `#FBF5EC` (light warm beige, between background and surface)
- Status cells: render as badges (see Section 3)
- Empty state: oversized lucide icon at 25-30% opacity, warm heading text below

### Empty/Welcome State

- Large lucide icon (48-64px) at 25-30% opacity
- Heading: Text `#292524`, semibold
- Subtext: Text Muted `#6D6560`
- Suggestion chips: Surface Elevated `#FFFFFF` background, Border `#D6D3D1` border, hover adds subtle shadow lift

## 7. Motion and Animation

| Animation | Duration | Easing | Details |
|-----------|----------|--------|---------|
| Panel slide | 200ms | ease-out | Slide in from right |
| Hover transitions | 150ms | ease | Background color, shadow changes |
| Shimmer (streaming) | 1.5s | linear, infinite | Subtle gradient sweep left-to-right |
| Streaming cursor | pulse | - | 1.5px wide, Primary `#B45309` color |
| Streaming indicator | - | - | min-height 24px to prevent CLS |

### Reduced Motion

All animations respect `prefers-reduced-motion: reduce`:
- Durations collapse to 0ms
- Shimmer and pulse animations stop
- Panel appears instantly (no slide)
- Streaming indicator still reserves space (min-height stays)

## 8. Accessibility Requirements

### Critical
- Color contrast: WCAG AA minimum 4.5:1 (verified in Section 1)
- Focus indicators: 2px solid Accent `#0369A1` with 2px offset on all interactive elements (blue, not amber - familiar clickable affordance)
- Touch targets: minimum 44x44px
- Icon-only buttons: `aria-label` required
- Color is never the sole indicator (badges always have text labels)

### High
- Body text minimum 15px (mobile and desktop)
- Line height 1.5-1.75 for body text
- Line length capped at 65-75 characters (`max-w-4xl`)
- No horizontal scroll on any viewport
- Loading feedback for operations over 300ms
- Error messages placed near the triggering element
- Every form input must have a visible, associated label

### Medium
- Pointer cursor on all clickable elements
- Consistent spacing scale (Section 4)
- Responsive breakpoints: 768px (tablet), 1024px (desktop)
- Animations 150-300ms (Section 6)
- Buttons disabled during async operations
- SVG icons only (no emoji as UI elements)
- Consistent icon sizing within context
- Reduced-motion support for all animations (Section 6)

## 9. Technical Setup

### Tailwind CSS v4 @theme Block

Replace the existing `@theme` block in `globals.css`:

```css
@theme {
  /* Core palette - warm earthy beige (Issue #13) */
  --color-primary: #B45309;
  --color-primary-soft: #FEF3C7;
  --color-secondary: #57534E;
  --color-accent: #0369A1;
  --color-background: #F8F0E3;
  --color-surface: #F0E9DC;
  --color-surface-elevated: #FFFFFF;
  --color-text: #292524;
  --color-text-muted: #6D6560;
  --color-border: #D6D3D1;
  --color-border-subtle: #E7E5E4;

  /* Semantic colors */
  --color-success: #16A34A;
  --color-warning: #D97706;
  --color-danger: #DC2626;

  /* Font references */
  --font-heading: var(--font-inter), -apple-system, BlinkMacSystemFont, sans-serif;
  --font-body: var(--font-inter), -apple-system, BlinkMacSystemFont, sans-serif;
}
```

This makes tokens available as Tailwind utilities: `bg-background`, `text-text-muted`, `border-border`, etc.

### Usage Rules
- Always use semantic token names in Tailwind classes (e.g., `bg-surface`, `text-text-muted`)
- Never hardcode hex values in component files (except StatusBadge - see Section 3 Token Architecture note)
- Update the WCAG table in Section 1 if any token value changes
- Status badge colors are component-local (Section 3), not global @theme tokens
- Use `bg-white` sparingly - prefer `bg-surface-elevated` with `border-border`

### Future Tokens
- `--color-accent-soft`: light accent tint for info callouts/backgrounds (add when needed by a component)
